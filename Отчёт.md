# Отчёт по тестированию геометрических модулей

## 1. Цели и задачи тестирования

**Цель тестирования** — проверка корректности вычисления площади и периметра
геометрических фигур.

**Задачи тестирования:**
- Проверить корректность математических вычислений.
- Проверить работу функций с нулевыми и положительными значениями.
- Подтвердить стабильность кода при автоматическом запуске тестов.

---

## 2. Описание тестируемого продукта

Тестируемый продукт — набор Python-модулей для вычисления параметров
геометрических фигур:

- `triangle.py`
- `circle.py`
- `rectangle.py`
- `square.py`

Каждый модуль содержит функции вычисления площади и периметра фигуры.

---

## 3. Область тестирования

В область тестирования входят:

### Треугольник (`triangle.py`)
- `area(a, h)`
- `perimeter(a, b, c)`

### Круг (`circle.py`)
- `area(r)`
- `perimeter(r)`

### Прямоугольник (`rectangle.py`)
- `area(a, b)`
- `perimeter(a, b)`

### Квадрат (`square.py`)
- `area(a)`
- `perimeter(a)`

Тестируются:
- положительные значения
- нулевые значения
- вещественные числа (для круга)

---

## 4. Стратегия тестирования

- **Тип тестирования:** модульное (Unit Testing)
- **Метод:** автоматизированное тестирование
- **Инструмент:** библиотека `unittest`
- **CI:** GitHub Actions

Тесты автоматически запускаются при каждом `push` в репозиторий.

---

## 5. Критерии приёмки

Продукт считается прошедшим тестирование, если:
- все unit-тесты выполняются без ошибок;
- результаты вычислений соответствуют ожидаемым;
- workflow GitHub Actions успешно выполняется на:
  - `ubuntu-latest`
  - `windows-latest`.

---

## 6. Ожидаемые результаты

- Успешное прохождение всех тестов
- Отсутствие дефектов в вычислениях
- Подтверждение качества кода
- Отчёт GitHub Actions со статусом **Success**
